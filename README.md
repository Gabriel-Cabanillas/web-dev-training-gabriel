#  web-dev-training-gabriel

**Full Stack Web Development Log.** *Project implementation, design patterns, and reinforcement of technical fundamentals.*

Este repositorio es mi laboratorio personal de aprendizaje y perfeccionamiento técnico. Aquí documento de forma estructurada mi camino hacia el dominio del desarrollo **Full Stack**, aplicando estándares de calidad profesional, clean code y arquitecturas escalables.

---

## 📁 Estructura del Repositorio

El proyecto está organizado en módulos que cubren desde los pilares de la web hasta integraciones complejas:

* **`01-javascript`**: Refuerzo de lógica, ES6+, asincronía y manipulación del DOM.
* **`02-typescript`**: Tipado avanzado, interfaces, genéricos y patrones aplicados con TS.
* **`03-html-css`**: Maquetación semántica, layouts complejos (Grid/Flexbox) y optimización de UI/UX.
* **`04-react`**: Hooks personalizados, gestión de estado y optimización de renderizado.
* **`05-apis-http`**: Diseño y consumo de APIs RESTful, protocolos HTTP y manejo de errores.
* **`06-supabase`**: Configuración de Backend-as-a-Service, autenticación y manejo de bases de datos relacionales.
* **`07-testing`**: Implementación de pruebas unitarias y de integración (Jest, Vitest, RTL).
* **`08-mini-kotta`**: Prototipado de funcionalidades específicas para el ecosistema de gestión de condominios.
* **`09-interview-prep`**: Algoritmos, conceptos teóricos clave y simulacros técnicos.

##  Estándares de Trabajo

Para mantener este repositorio con un nivel de calidad **Senior**, aplico estrictamente:

* **Conventional Commits**: Uso de prefijos estandarizados (`feat:`, `fix:`, `docs:`, `refactor:`, `test:`, `chore:`) para un historial de cambios legible.
* **TypeScript First**: Priorizando el tipado fuerte para garantizar la robustez del código.
* **Documentation-Driven**: Cada módulo cuenta con su propio archivo de notas detallando el proceso de resolución.

---

##  Reglas del Entrenamiento (Protocolo de Aprendizaje)

Para garantizar independencia técnica y un dominio real de las herramientas, me rijo por estas tres reglas obligatorias:

1.  **Cero IA al primer intento:** Prohibido delegar la lógica inicial a herramientas generativas. Primero resuelvo, busco documentación oficial y depuro errores manualmente. La IA se utiliza únicamente para revisión o optimización posterior.
2.  **Enfoque en la Explicación:** Un módulo no se considera terminado cuando el código funciona, sino cuando soy capaz de explicar el "por qué" detrás de cada línea y arquitectura.
3.  **Bases sobre Portafolio:** No diseño portafolios bonitos sobre bases flojas. El enfoque aquí es la excelencia técnica y la lógica de programación pura.
